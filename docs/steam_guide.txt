[h1]SE.PB Code Generator — Visual Programmable Block Scripting[/h1]

[i]Build C# scripts for the Space Engineers Programmable Block by clicking — no coding required.[/i]

A free, offline-capable web tool that turns drag-and-drop logic into working PB scripts. Sensors, doors, batteries, thrusters, LCDs and 48 more block types — pick what to check and what to do, copy the generated code into your Programmable Block, done.

[h2]Quick Start (60 seconds)[/h2]

[olist]
[*]Open the [url=https://head87x.github.io/se-pb-codegen/]live version[/url] in your browser.
[*]Left panel: drag a block from the [b]BLOCKS[/b] palette into the [b]IF / THEN / ELSE[/b] area, or click [b]+ Condition[/b] / [b]+ Action[/b].
[*]Middle panel: pick what each block should check or do (e.g. Sensor → "Detected something" → Door → "Open").
[*]Right panel: copy the generated C# code with [b]⧉ Copy[/b].
[*]In Space Engineers: Programmable Block → [b]Edit[/b] → paste → [b]Check Code[/b] → [b]Remember & Exit[/b].
[/olist]

[h2]What it does[/h2]

[list]
[*][b]52 block types[/b] in 12 categories. All practical conditions and actions per block — not just the obvious ones.
[*][b]Drag-and-drop block palette[/b] on the left. Search field, category folding.
[*][b]Nested conditions[/b] with AND / OR, plus optional [b]ELSE[/b] branch.
[*][b]Block groups[/b] — toggle per condition or action to apply it to a whole SE block group ("all front lights on", "any sensor triggered").
[*][b]Inventory checks[/b] on 9 block types — "has > X iron ore", "has < Y uranium", etc., with a styled subtype dropdown for ores / ingots / components / ammo / tools.
[*][b]5 execution modes[/b]: manual (argument / button), continuous, Update1 / Update10 / Update100.
[*][b]Tooltips[/b] on every option — short explanation of what it does and what values are sensible.
[*][b]Live code preview[/b]: the generated C# updates as you type, no fokus loss.
[/list]

[h3]Multi-LCD layout (Phase 5)[/h3]

Multiple adjacent LCDs can be treated as [b]one virtual display[/b]. Pick columns and rows (1–6 each), give a name pattern ([b]LCD {col}{row}[/b] → LCD A1, LCD B1, LCD A2, LCD B2), and place widgets anywhere on the combined canvas. The generator emits a [b]for-loop[/b] over all physical LCDs; each renders only its slice. Widgets that straddle two LCDs are drawn on both — SE clips off-screen sprites automatically.

[*]Check: [b]Detected something[/b]
[/list]
[*][b]THEN[/b] → + Action
[list]

[*]Right panel shows the C# code. Copy → paste into PB → done.

[h2]Block groups[/h2]

Each condition and action has a toggle "[b]Apply to a block group[/b]". When enabled, the name field becomes a [b]group name[/b] (matching an SE block group in the terminal):

[list]
[*][b]Action on group[/b] runs for every block in the group ([i]foreach[/i] loop in the generated code). Example: "all front lights → on".
[*][b]Condition on group[/b] is true as soon as [b]any[/b] block in the group meets it (LINQ [i].Any()[/i]). Example: "any reactor below 50% fuel".
[/list]

This skips the tedious "add 8 conditions / 8 actions" routine for fleet-wide patterns.

[hr][/hr]

[h2]Custom mode[/h2]

If something isn't in the catalogue, pick block type [b]Custom (manual entry)[/b]. You get fields for:

[list]
[*][b]ApplyAction("...")[/b] — any in-game terminal action by name
[*][b]SetValueBool / SetValueFloat[/b] — for properties not exposed directly
[*][b]Raw C# code[/b] with [b]{v}[/b] as a placeholder for the block variable
[/list]

So custom needs like [code]{v}.Enabled = !{v}.Enabled;[/code] (toggle) or [code]((IMyAssembler){v}).Mode = MyAssemblerMode.Disassembly;[/code] (disassembly mode) are one input field away.

[hr][/hr]

[h2]Tips[/h2]

[list]
[*][b]Block names must match the terminal exactly[/b] (case, spaces, special characters). In-game: select the block, copy the name from the field at the top of the terminal.
[*][b]C# evaluates && before ||[/b]. For complex bracketing, use Custom mode with a raw expression.
[*][b]Empty IF[/b] means "always true" — actions run unconditionally, useful for constant outputs (status LCD, gauges).
[*][b]Multi-LCD widgets can span borders[/b] — drag them across the dashed separator lines in the preview, the generator handles the offsets.
[*][b]LCD format[/b] only affects the preview aspect. The generated code reads the [i]actual[/i] LCD format at runtime via [b]SurfaceSize[/b], so the same script works on Square, Wide, Tall and Big LCDs.
[/list]
