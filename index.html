▶ BLÖCKE [ DRAG ]
Block ins WENN/DANN/SONST ziehen, oder weiter "+ Bedingung/Aktion" verwenden.
▶ BAUKASTEN [ INPUT ]
::: VORLAGEN
Keine Vorlagen gespeichert.
::: SHARE-TOKEN
Speichert / lädt deine komplette Konfiguration (Bedingungen, Aktionen, LCD-Baukasten) als kopierbaren Token. Token-Inhalt liegt direkt in der Zeichenkette — kein Server, kein Account.
::: SKRIPT-INFO (Workshop)
Optional: trägt Workshop-Metadaten in den Header des generierten Codes ein. Der Attribution-Hinweis auf das Tool bleibt immer erhalten — diese Felder sind zusätzlich.
::: AUSFÜHRUNG
Verteilt LCD-Updates über mehrere Ticks. Empfohlen ab Multi-LCD 3×3 oder vielen Aggregator-Widgets — vermeidet „Script too complex"-Fehler. Trade-off: LCDs aktualisieren nicht alle gleichzeitig, sondern ein Display pro Tick.
::: WENN (Bedingungen)
Keine Bedingungen — Aktion läuft immer.
::: DANN (Aktionen)
Noch keine Aktionen.
::: SONST (optional)
Keine SONST-Aktionen.
::: LCD STATUS-AUSGABE (Text)
Einfache Text-Ausgabe: schreibt Status, Bedingungen und Aktionen aufs LCD.
::: LCD BAUKASTEN (Grafik)
PB hat meist nur Index 0. Cockpits: 0–7 je nach Modell.
Platzhalter: {col} = A, B, C, … · {row} = 1, 2, 3, … · {c}/{r} = numerisch (1, 2, 3, …)
Widgets per Maus in der Vorschau positionieren und skalieren. Der generierte C#-Code nutzt zur Laufzeit das echte LCD-Format aus dem Spiel.
